Prueba la API de e-coordina

Valida accesos enviando centro, empresa y DNI; el resto de datos quedan como referencia.

Datos obligatorios: centro, CIF y DNI Valores por defecto desde el backend Respuesta detallada

Recordatorio

Solo se envían centro, CIF y DNI. Los datos personales se quedan en local.

Webservice

Probar API e-coordina

Lanza una petición al servicio https://ws.e-coordina.com/1.4 con los datos mínimos (centro, CIF y DNI). El nombre y los apellidos se guardan solo a nivel local y no viajan en la consulta. 


          

            

              Según la ayuda oficial (documentación),
              los parámetros obligatorios son user, token, instance, los formatos
              in/out (por defecto json) y el action_type que debe ser do para enviar datos;
              el action solicitado aquí es acceso_permitido_data.
            

            
Ejemplo de cuerpo esperado según la versión 1.4 del servicio:

            
{
  "user": "webservice",
  "token": "TU_TOKEN",
  "instance": "elecnor",
  "in": "json",
  "out": "json",
  "action_type": "do",
  "action": "acceso_permitido_data",
  "data": {
    "centro_cod": "512/73000-A79408683",
    "empresa_cif": "B45740816",
    "trabajador_dni": "70590602W"
  }
}

            
Si la API responde con status: "ERROR" revisa la acción, credenciales y parámetros enviados.
